FAQ

Preguntas frecuentes

¿Cuánto tiempo lleva desarrollar una aplicación a medida?
Depende del alcance. Un MVP funcional puede estar listo en 6-8 semanas. Una aplicación completa con integraciones puede llevar entre 3 y 6 meses. Después del Discovery te presentamos un cronograma con hitos claros y fechas de demo quincenales.
¿Pueden integrarse con los sistemas que ya uso (CRM, ERP, pasarelas de pago)?
En la mayoría de los casos sí. Si tu sistema tiene API (HubSpot, Salesforce, MercadoPago, Stripe, SAP u otras), podemos conectarnos. Si no tiene API, evaluamos acceso directo a base de datos u otras alternativas. Lo determinamos en el Discovery.
¿Qué pasa si los requerimientos cambian durante el proyecto?
Trabajamos en sprints quincenales. Los cambios pueden incorporarse en el sprint siguiente o priorizarse según su impacto. No congelamos el scope ni paralizamos el proyecto ante cambios: los gestionamos con vos en cada ciclo.
¿El hosting y la infraestructura están incluidos?
No automáticamente. La infraestructura (servidores, dominios, cuentas cloud) es un costo separado que va directo a las plataformas (Vercel, Railway, AWS, etc.). Te asesoramos en la elección y, si querés, lo gestionamos nosotros. Se define en la propuesta.
¿Pueden agregar IA a un software que ya tenemos?
Sí. Podemos integrar agentes IA, LLMs o automatizaciones en sistemas existentes, siempre que el código sea accesible o el sistema tenga API. Lo evaluamos en el Discovery. El módulo de IA es opcional �?" no es obligatorio en todos los proyectos.
¿Cómo se cotiza un proyecto de software a medida?
Después del Discovery (gratuito) presentamos una propuesta con alcance, tecnología, cronograma y presupuesto. El precio depende de la complejidad funcional, las integraciones necesarias y el tiempo estimado. No hay precio fijo: cada proyecto es diferente.
¿El código fuente es nuestro?
Sí, el código fuente es tuyo desde el día uno. Trabajamos en tu repositorio o en uno que te entregamos con acceso completo. Al finalizar el proyecto entregamos documentación técnica para que tu equipo o cualquier otro desarrollador pueda continuar el trabajo.
¿Qué tecnologías usan y por qué?
Preferimos un stack mantenible a largo plazo: Next.js o Astro para frontend, NestJS o FastAPI para backend, PostgreSQL como base de datos principal, Railway o Vercel para deploy. La elección depende del caso �?" no usamos herramientas por moda sino por adecuación al problema.
